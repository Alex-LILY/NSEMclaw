│ 优化 #2: 写入队列异步化                                          │
├─────────────────────────────────────────────────────────────────┤

│ 问题: 同步写入SQLite阻塞主线程，降低摄入吞吐量                  │
│                                                                 │
│ 解决方案:                                                       │
│ ┌─────────────────────────────────────────────────────────────┐ │

│ │   private queue: WriteTask[] = [];                          │ │
│ │   private flushInterval = 100; // ms                        │ │
│ │   private batchSize = 1000;                                 │ │
│ │                                                             │ │
│ │   enqueue(task: WriteTask) {                                │ │
│ │     this.queue.push(task);                                  │ │
│ │     if (this.queue.length >= this.batchSize) {              │ │
│ │       this.flush();                                         │ │
│ │     }                                                       │ │
│ │   }                                                         │ │
│ │                                                             │ │
│ │   async flush() {                                           │ │
│ │     const batch = this.queue.splice(0, this.batchSize);     │ │
│ │     await sqlite.insertBatch(batch); // 单次事务            │ │
│ │   }                                                         │ │

┌─────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ 问题: 全局 withLock 导致查询和摄入互斥，无法并发                │
│                                                                 │
│ 解决方案:                                                       │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │ class ReadWriteLock {                                       │ │
│ │   private readers = 0;                                      │ │
│ │   private writer = false;                                   │ │
│ │   private waitQueue: Array<() => void> = [];                │ │
│ │                                                             │ │
│ │   async readLock<T>(fn: () => T): Promise<T> {              │ │
│ │     await this.acquireRead();                               │ │
│ │     try { return fn(); }                                    │ │
│ │     finally { this.releaseRead(); }                         │ │
│ │   }                                                         │ │
│ │                                                             │ │
│ │   async writeLock<T>(fn: () => T): Promise<T> {             │ │
│ │     await this.acquireWrite();                              │ │
│ │     try { return fn(); }                                    │ │
│ │     finally { this.releaseWrite(); }                        │ │
│ │   }                                                         │ │
│ │ }                                                           │ │
│ │                                                             │ │
│ │ // activate() 使用 readLock                                  │ │
│ │ // ingest() 使用 writeLock                                   │ │
│ │ // 多个activate可并行，ingest排他                             │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ 预期收益: 并发查询能力 1 → N (取决于CPU核心数)                  │

### 8.3 最终建议

**对于 128GB + RTX 4090 环境:**

1. **立即实施 (本周)**
   - 批量加载和异步写入可显著提升用户体验
   - 改动风险低，收益明显

2. **短期投资 (本月)**
   - GPU加速搜索充分利用4090的算力
   - ANN索引为未来扩展打基础

3. **长期规划 (Q3-Q4)**
   - 分布式存储应对Agent数量增长
   - 智能预加载提升AI助手的"直觉"能力

**监控指标:**

- 目标缓存命中率: >90%
- 目标查询P99延迟: <100ms
- 目标摄入吞吐量: >50/sec

## 🔌 对外接口

### UnifiedCoreV2 公共 API

```typescript
class UnifiedCoreV2 {
  // 生命周期
  initialize(config): Promise<void>
  shutdown(): Promise<void>
  
  // 记忆管理
  ingest(item: UnifiedMemoryItem): Promise<UnifiedMemoryItem>
  retrieve(query: string, options?: RetrieveOptions): Promise<UnifiedMemoryItem[]>
  access(id: string): Promise<UnifiedMemoryItem | null>
  
  // 会话管理
  startSession(userId: string, metadata?: object): string
  recordMessage(sessionId: string, message: Message): void
  recordToolCall(sessionId: string, toolCall: ToolCall): void
  endSession(sessionId: string): Promise<void>
  
  // 提取
  extractFromSession(sessionId: string): Promise<CandidateMemory[]>
  extractManually(content: string, context: string): Promise<CandidateMemory[]>
  
  // 兼容层
  createSearchManagerAdapter(): MemorySearchManager
  
  // 状态
  getStatus(): UnifiedCoreV2Status
}
```

## 🎨 架构优势

### 清理前 vs 清理后

| 对比项 | 清理前 | 清理后 |
|-------|-------|-------|
| **入口数量** | 4+ (Adapter 分散) | 1 (UnifiedCoreV2) |
| **文件数量** | ~60 个 | ~30 个 |
| **存储系统** | 3+ (NSEM2/NSEM21/UnifiedNSEM2) | 1 (UnifiedCoreV2 内部管理) |
| **会话管理** | 2 (Agent Runner + SessionManager) | 1 (SessionManager 作为模块) |
| **配置复杂度** | 高（多个配置体系） | 低（统一配置） |
| **数据流清晰度** | 低（多层转换） | 高（直达核心） |

### 核心改进

1. **单一职责**：每个模块只做一件事
2. **统一入口**：所有操作通过 UnifiedCoreV2
3. **分层清晰**：应用层 → 核心层 → 模块层 → 基础设施层
4. **可测试性**：模块间解耦，便于单元测试
5. **可扩展性**：新功能以模块形式接入

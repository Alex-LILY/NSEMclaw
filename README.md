## 📖 项目简介

**NSEM·瑶姬**（简称 NSEMclaw）是基于 **OpenClaw** 二次开发的多通道AI网关系统，是一个面向未来的下一代AI复合智能体架构。

2026年3月，NSEM 正式升级为 **NSEM 3.0 Phoenix（浴火重生版）**，带来了革命性的认知能力和全新的系统架构。

---

## ✨ 核心特性

### 🤖 智能体管理

- **多智能体并行运行**：支持同时运行多个独立的AI智能体
- **Subagent 协作模式**：主智能体可以派发子任务，实现复杂工作流程
- **人格系统**：支持多个人格切换（Alex / Alex-Lily）
- **弹性子代理编排**：断路器、重试机制、死信队列、超时控制

### 2. 智能决策系统 (SmartDecisionService)

NSEM 3.0 Phoenix 全新引入的智能决策系统，负责分析用户输入并决定路由方向：

```
用户输入
    │
    ▼
┌─────────────────────────┐
│ 1. 内容类型识别        │
│ - 检测图片/文本/文件   │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 2. 情感 & 上下文分析   │
│ - EmotionalIntelligence│
│ - 检测紧急程度、情绪   │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 3. 意图识别            │
│ - 关键词模式匹配       │
│ - taskType 推断        │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 4. 路由决策            │
│ route_to_vision/llm/tool│
└─────────────────────────┘
```

**意图识别规则：**

| 模式 | 任务类型 | 说明 |
|------|----------|------|
| `/提取.*文字\|OCR\|文字识别/` | `ocr` | 文字提取 |
| `/界面\|UI\|按钮\|布局/` | `ui` | UI分析 |
| `/代码\|程序\|函数/` | `code` | 代码分析 |
| `/是什么\|看看\|分析/` | `describe` | 通用描述 |

### 3. 视觉系统 (Vision System)

支持图片理解，按需加载模型：

```
用户发送图片
    │
    ▼
┌─────────────────────────┐
│ 1. 输入检测层          │
│ - 检测图片附件         │
│ - 提取图片路径        │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 2. 意图识别层          │
│ - 分析文字关键词       │
│ - 确定 taskType        │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 3. 视觉模型处理        │
│ - 按需加载 (~5-10秒)   │
│ - 构建 Prompt           │
│ - 分析图片              │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 4. 结果整合            │
│ 视觉结果 + 用户问题    │
│ → LLM 生成最终回复     │
└─────────────────────────┘
           │
           ▼
    启动5分钟空闲定时器
    → 自动卸载模型释放GPU
```

### 4. 记忆摄入流程 (Ingestion Pipeline)

```
输入内容
    │
    ▼
┌─────────────────────────┐
│ 1. 文本分块            │
│ ~400 tokens/块         │
│ 80 tokens 重叠         │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 2. 向量化               │
│ SmartEmbeddingEngine   │
│ 384维向量输出          │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 3. 重要性评分          │
│ ImportanceScorer        │
│ 10个维度, 0-1分        │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 4. 重复检查 & 内存管理 │
│ 相似度>0.95→强化      │
│ LRU淘汰                │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 5. 创建 & 存储          │
│ → MemAtom              │
│ → LRU Cache / Disk    │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 6. 建立关系 & 更新场   │
│ establishRelations()   │
│ updateFields()         │
└─────────────────────────┘
```

### 5. 记忆激活流程 (Activation Engine)

```
查询输入
    │
    ▼
┌─────────────────────────┐
│ 1. 查询扩展             │
│ QueryExpansion         │
│ 扩展5个相关查询        │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 2. 混合搜索             │
│ 向量 + BM25 + 时间衰减 │
│ 加权合并 (nsem*1.0 + trad*0.9)│
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 3. 缓存命中 & Fallback │
│ LRU Cache → Disk      │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 4. 重排序               │
│ Reranker (Cross-Encoder)│
│ Top-100 → Top-10       │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 5. MMR 多样性重排      │
│ 平衡相关性与多样性     │
└──────────┬──────────────┘
           │
           ▼
┌─────────────────────────┐
│ 6. 激活传播 & 更新     │
│ spreadActivation()     │
└──────────┬──────────────┘
           │
           ▼
    返回 ActivatedMemory
```

### 6. 记忆进化流程 (Evolution Engine)

定期执行的记忆维护任务：

| 任务 | 频率 | 说明 |
|------|------|------|
| `decay` | 每小时 | 记忆强度衰减 |
| `prune` | 每日 | 删除低于阈值的记忆 |
| `merge-fields` | 每日 | 合并相似记忆场 |
| `optimize-storage` | 每周 | 存储优化 |
| `rebuild-index` | 每月 | 重建索引 |

## 📦 AI 模型支持

NSEM 3.0 Phoenix 支持多种本地和云端 AI 模型：

### 1. 本地模型 (GPU 加速)

| 模型类型 | 模型名称 | 用途 | 大小 |
|----------|----------|------|------|
| **Embedding** | `hf_ggml-org_embeddinggemma-300M-Q8_0.gguf` | 文本向量化 | 329MB |
| **Expansion** | `hf_tobil_qmd-query-expansion-1.7B-q4_k_m.gguf` | 查询扩展 | 1.2GB |
| **Reranker** | `hf_ggml-org_qwen3-reranker-0.6b-q8_0.gguf` | 结果重排 | 639MB |
| **视觉模型** | Llava-Phi-3-mini | 图片分析 | 本地加载 |

### 2. 云端模型 (API)

| 提供商 | 模型 | 说明 |
|--------|------|------|
| **MiniMax** | M2.5 | 主语言模型 |
| **OpenAI** | GPT-4o | 通用对话 |
| **Anthropic** | Claude 3.5 | 通用对话 |
| **Google** | Gemini | 多模态 |
| **Ollama** | 本地模型 | 本地推理 |

### 3. 资源模式

| 模式 | 内存占用 | 加载模型 |
|------|---------|---------|
| **minimal** (极简) | ~314MB | embedding |
| **balanced** (平衡) | ~924MB | embedding + reranker |
| **performance** (性能) | ~2.2GB | 全部三个模型 |

## 🗃️ 三层记忆存储

```
判断存储层级 (基于访问频率和重要性)
        │
        ▼
┌─────────────────────────────────────────────────┐
│           热数据层 (Hot / Working)             │
│  ┌─────────────────────────────────────────┐   │
│  │ LRU Cache                                │   │
│  │ maxSize: 15 atoms                       │   │
│  │ 访问速度: ~1ms                          │   │
│  └─────────────────────────────────────────┘   │
└────────────────────┬────────────────────────────┘
                     │ 超过容量时淘汰
                     ▼
┌─────────────────────────────────────────────────┐
│           温数据层 (Warm / Short-Term)          │
│  ┌─────────────────────────────────────────┐   │
│  │ Map<string, MemAtom> + 内存索引         │   │
│  │ maxSize: 1000 atoms, TTL: 24小时        │   │
│  │ 访问速度: ~10ms                         │   │
│  └─────────────────────────────────────────┘   │
└────────────────────┬────────────────────────────┘
                     │ TTL过期或访问频率低
                     ▼
┌─────────────────────────────────────────────────┐
│           冷数据层 (Cold / Long-Term)          │
│  ┌─────────────────────────────────────────┐   │
│  │ SQLite + 文件系统                        │   │
│  │ 存储上限: 无限                          │   │
│  │ 访问速度: ~100ms                        │   │
│  └─────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

| 记忆层 | 容量 | 特点 |
|--------|------|------|
| **工作记忆 (Hot)** | 15条 | LRU管理，快速访问 |
| **短期记忆 (Warm)** | 1000条 | 可配置TTL，语义检索 |
| **长期记忆 (Cold)** | 磁盘 | 永久存储，跨会话 |

## 🔄 完整调用链

```
用户发送图片消息
    │
    ▼
1. Gateway 接收 - 解析消息/提取附件
    │
    ▼
2. InputProcessor - 检测类型/构建消息
    │
    ▼
3. SmartDecisionService - 内容识别→情感分析→意图识别→路由决策
    │
    ▼
4. VisionModelEngine - 按需加载模型→构建Prompt→执行推理→返回描述
    │
    ▼
5. NSEM.activate() - 查询扩展→混合搜索→重排序→激活记忆
    │
    ▼
6. LLM (MiniMax M2.5) - 整合视觉结果+记忆→生成回复
    │
    ▼
7. Output Formatter - 格式化输出/发送给用户
    │
    ▼
8. NSEM.ingest() - 摄入对话到记忆
    │
    ▼
等待下一次交互
```

## 🚀 快速开始

### 环境要求

- Node.js >= 20.x
- pnpm >= 8.x
- Linux/macOS

### 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/Alex-LILY/NSEMclaw.git
cd NSEMclaw

# 2. 安装依赖
pnpm install

# 3. 配置环境
cp nsemclaw.config.example.json nsemclaw.json

# 4. 启动
pnpm start
```
